### Задание 1

`1.1. Восстановление данных за предыдущий день

Полный бэкап (ежедневно, например, в полночь) + инкрементальный бэкап (в течение дня).
Почему: Полный бэкап восстанавливает базу на начало дня, инкрементальный — изменения в течение дня.
Инструменты: PostgreSQL (pg_dump/pg_basebackup), MySQL (mysqldump/Percona XtraBackup), SQL Server (BACKUP DATABASE).


1.2. Восстановление данных за час до поломки

Полный бэкап + WAL-логи (или журналы транзакций) для Point-in-Time Recovery (PITR).
Почему: WAL-логи позволяют восстановить базу до точного момента (например, за час до сбоя).
Инструменты: PostgreSQL (pg_basebackup + archive_mode), MySQL (Percona XtraBackup + binary logs), SQL Server (BACKUP LOG).`

### Задание 2

`Резервное копирование (pg_dump)

pg_dump -U username -d dbname -F c -f backup_file.dump


Восстановление (pg_restore)

pg_restore -U username -d dbname --create --verbose backup_file.dump`
